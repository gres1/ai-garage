# SPEC: «Live view» — стена живых превью (интегрировать в web + desktop)

## Что это
Новый РЕЖИМ в AI Garage: выбранные сервисы показываются как ЖИВЫЕ интерактивные `iframe` рядом в сетке, на весь экран. Рабочий прототип готов и проверен — **`public/multi.html`**. Читай его первым делом: там точные CSS/JS, копируй оттуда.

## ⚠️ Что в прототипе НЕ так — ОБЯЗАТЕЛЬНО исправить при интеграции
1. **Детект блокировки глючит — переделать.** Источник правды = **`/api/can-embed?port=PORT`** (same-origin в основной панели — работает надёжно). Не использовать наивный таймаут как признак блокировки:
   - **OpenClaw (:18789)** показывает РОДНУЮ ошибку браузера («Сайт localhost не позволяет соединение») вместо оверлея — в standalone-прототипе can-embed недоступен (CORS). В панели (same-origin) can-embed вернёт `embeddable:false` → показать оверлей. ОБЯЗАТЕЛЬНО проверить, что OpenClaw показывает оверлей, а не битый вид.
   - **Hermes (:9119)** грузится МЕДЛЕННО, но мой 5-сек таймаут ложно метит его «заблокирован». Так нельзя. Правило: если `can-embed` → `embeddable:true`, держать спиннер «Загрузка…» сколько надо (не метить blocked по таймауту). «blocked» ставить ТОЛЬКО когда `can-embed` сказал `false`.
2. **Логотип** — НЕ оставлять текущий значок (Az называет его «беспонтовый», это placeholder `logo.png`/emoji). Поставить **настоящий бренд-лого AI Garage** (Az даст файл/ассет). 
3. **Название режима** — «**Live view**» (или согласовать с Az: «Все экраны»/«Витрина»), НЕ «Wall».
4. **«Пикчер»/«Services»** — назвать по-человечески: «**Сервисы**» / «Выбор сервисов» (это выпадашка с чекбоксами — какие сервисы показать на стене).

## Куда добавить
- В основной **`public/index.html`** как переключаемый режим (кнопка в шапке «Список ⇄ Live view» или роут `#live`). **Один порт 7777**, не отдельная страница/порт для пользователя (так понятнее).
- **Десктоп (Tauri)** использует тот же `index.html` → после правки **пересобрать Tauri** (`desktop/src-tauri`), режим появится и в `.app`. Отдельно дублировать не нужно.

## Требования (всё реализовано в прототипе `multi.html`)
1. **Сетка живых iframe** выбранных сервисов; переключатель колонок **1/2/3/4**.
2. **Зум** — по умолчанию **75%**, кнопки **−/+** (40–150%). Реализация: на iframe `transform:scale(var(--zoom)); transform-origin:top left; width/height:calc(100%/var(--zoom))`, у контейнера `.frame{overflow:hidden}`. (Это решает «сайт увеличен, формы не влезают».)
3. **Fullscreen** — кнопка-**тоггл** (`requestFullscreen`/`exitFullscreen` + слушатель `fullscreenchange` меняет подпись на «Exit»). Должна СВОРАЧИВАТЬСЯ обратно.
4. **Пикчер сервисов** — какие сервисы на стене (чекбоксы, состояние в `localStorage`). **Z-index выше iframe**: шапка `position:relative; z-index:100`, меню `z-index:200` — иначе iframe режет выпадашку.
5. **Заблокированный сервис** — НЕ показывать нативную ошибку браузера («Сайт localhost не позволяет соединение»). Спросить **`/api/can-embed?port=PORT`** → если `{embeddable:false}` → показать оверлей с уже существующими i18n-строками `blockedTitle`/`blockedBody` («Preview blocked by the service / This service doesn't allow being shown inside a frame — that's its own security setting, not a panel error»). Плюс спиннер «Загрузка…» пока iframe грузится. (В прототипе can-embed зовётся, но standalone на 7790 его не достаёт из-за CORS — при интеграции в панель он same-origin и работает.)
6. **Плитка**: шапка (имя, статус-точка, `open ↗`, `✕` убрать со стены).
7. **Логотип** `logo.png` вместо emoji; название режима — **«Live view»** (либо согласовать с Az: «Все экраны» / «Витрина» / «Превью»).
8. **i18n** всех строк (минимум en/ru), как в основном app.
